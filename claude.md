# vimember 项目全景文档

> 本文档用于帮助 AI 快速理解 vimember。它不是产品最终规格，遇到冲突时以 `AGENTS.md`、用户最新指令和用户指定读取的 Figma 内容为准。

## 1. 项目概述

vimember 是一个计划中的 iOS 视频日记 App。

核心流程：

1. 用户从系统相册导入视频。
2. App 将视频复制到本地沙盒。
3. 用户进入编辑页并为视频添加文字。
4. 用户结束编辑，保存为一条视频日记。
5. 用户可以在首页列表或画廊视图浏览视频日记摘要。
6. 用户可以点进详情页查看完整视频和完整文字。

当前阶段只做文档整理和计划制定，不开始制作 App。

## 2. 已明确需求

### 2.1 视频日记

每条日记由一个视频和一段文字组成。

已确认：

1. 视频文件复制到 App 沙盒，由 App 使用本地文件播放和展示。
2. 文字保存到 App 本地数据库。
3. 可以保存原相册视频来源标识，但不能依赖它作为唯一播放来源。
4. 最低支持 iOS 17.0。

### 2.2 首页视图

首页展示所有视频日记，上下排列。视频和搭配文字放在一起，只显示文字摘要。点击可进入详情页。

首页滚动行为：

1. 上下滑动时视频不动。
2. 停止滑动后，当前屏幕中最主要的一条可见视频立刻开始播放。
3. 同一时间只播放一条视频。

### 2.3 画廊视图

画廊展示所有视频日记，只显示文字摘要。点击可进入详情页。

画廊具体视觉形式待 Figma 指定。

### 2.4 详情视图

详情页展示完整视频和完整文字。视频自动播放并循环播放。详情页支持再次编辑和删除，具体入口、样式和确认流程待 Figma 指定。

### 2.5 删除规则

删除只删除 App 本地数据库记录和 App 沙盒中的视频文件，任何时候不删除系统相册原视频。

### 2.6 未来版本

第二版可能加入视频分类功能。第一版不实现分类 UI 和分类流程，但数据结构设计应避免明显阻碍后续扩展。

### 2.7 GitHub 版本管理

项目使用 GitHub 做版本管理。`main` 分支用于稳定版本，开发工作默认使用 `codex/` 前缀功能分支，完成后通过 PR 合并。

### 2.8 Figma 优先

## 4. 建议技术分层

以下是后续可能采用的分层方向，正式实现前仍需结合用户确认和实际工程状态调整。

### App

应用入口、环境配置、权限描述和根导航。

### Features

按功能分区：

1. Import：相册视频导入。
2. Editor：视频文字编辑。
3. Home：纵向首页列表。
4. Gallery：画廊。
5. Detail：详情页。

### Domain

视频日记数据模型、摘要规则、播放状态规则。

### Services

相册访问、视频资源管理、缩略图生成、持久化、权限。

### DesignSystem

来自 Figma 的颜色、字体、间距、组件和图标。

## 5. 关键风险

1. 本地复制视频会占用手机存储，需要后续设计容量提示和清理策略。
2. 首页视频自动播放会影响滚动性能和电量。
3. “最主要的一条视频”的判定需要结合 Figma 和用户确认。
4. Figma 未读取前实现 UI 容易偏离用户设计。
5. 摘要字数和截断样式未定，会影响首页和画廊的信息密度。
6. 详情页循环播放需要正确释放播放器资源。

## 6. 待确认问题

1. 摘要固定字数、行数和截断样式。
2. 首页“最主要的一条视频”的具体判定方式。
3. 画廊布局。
4. 再次编辑按钮位置和样式。
5. 删除入口、确认流程和视觉样式。
6. 视频是否静音自动播放，以及是否显示播放控制条。
7. 是否需要浅色和深色模式。
8. 搜索、分享、导出、账号、同步、备份等后续能力是否进入未来版本。
