## 2. 当前结论

vimember 是一个新项目，不能继承旧项目的照片记忆语义和页面结构。

当前已明确：

1. App 类型：iOS 视频日记 App。
2. 输入：从相册导入视频。
3. 存储：视频复制到 App 沙盒，文字保存到 App 本地数据库。
4. 编辑：导入后进入编辑页，为视频添加文字，后续支持再次编辑。
5. 首页：上下排列视频日记，只显示摘要，停止滚动后只播放最主要的一条可见视频。
6. 画廊：展示视频日记，只显示摘要，点击进入详情。
7. 详情：完整视频和完整文字，视频循环自动播放。
8. 删除：只删除 App 本地记录和 App 沙盒视频文件，不删除系统相册原视频。
9. 技术：最低支持 iOS 17.0，优先 SwiftUI + SwiftData。
10. 版本管理：使用 GitHub，`main` 稳定分支，`codex/` 开发分支。
11. 未来版本：第二版可能加入视频分类功能，第一版不实现。
12. 视觉来源：用户指定的 Figma 内容。

## 3. 当前主要风险

### 高风险

1. Figma 尚未读取具体节点。
   - 任何 UI 实现都可能偏离用户已经绘制好的设计。

2. 本地复制视频会增加存储占用。
   - 需要后续考虑容量提示、删除清理和失败恢复。

3. 首页“最主要的一条视频”的判定方式未确认。
   - 影响播放策略、滚动检测和性能。

4. 摘要具体字数和样式未确认。
   - 首页和画廊的信息密度无法稳定实现。

### 中风险

1. 再次编辑按钮位置未确认。
2. 删除入口和确认流程未确认。
3. 是否需要深色模式未确认。
4. 视频是否静音自动播放未确认。
5. GitHub 仓库尚未创建，远端分支策略尚未落地。
