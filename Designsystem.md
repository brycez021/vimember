## 1. 设计来源

正式 UI 的唯一视觉来源是用户提供的 Figma 文件中用户指定读取的内容：

## 4. 视频相关视觉规则

已明确的产品行为：

1. 首页视频和文字摘要必须放在一起。
2. 画廊只展示摘要。
3. 详情页展示完整视频和完整文字。
4. 详情页视频循环自动播放。
5. 首页滚动停止后，只播放当前屏幕中最主要的一条可见视频。
6. 导入视频后必须进入编辑页。
7. 详情页支持再次编辑和删除，但入口位置与视觉样式由 Figma 指定。
8. 删除不影响系统相册原视频。

待确认的视觉细节：

1. 视频比例。
2. 视频圆角。
3. 封面图截取方式。
4. 播放中是否显示控制条。
5. 摘要字数和行数。
6. 画廊卡片比例和列数。
7. 详情页文字排版。
8. 再次编辑按钮样式和位置。
9. 删除入口、确认流程和视觉样式。

## 5. 维护规则

1. 不得为了方便实现而改变 Figma 视觉意图。
2. 不得从旧项目参考文件继承颜色、字体或布局。
3. 新增 Token 必须写入本文件。
4. 修改 Token 必须说明原因和影响范围。
5. 如果 Figma 与代码实现能力存在冲突，先向用户说明，再等待确认。
