# vimember

vimember 是一个计划中的 iOS 视频日记 App。核心体验是：从系统相册导入视频，为视频添加文字，然后在首页列表、画廊和详情页中回看这些视频日记。

当前仓库阶段：只完成指导文件整理和开发计划，尚未开始制作 App。

## 产品目标

vimember 要解决的是一种轻量的视频记忆记录需求：

1. 从相册选择或导入视频。
2. 导入后进入编辑页。
3. 给视频写一段文字。
4. 完成编辑，保存为视频日记。
5. App 使用复制到本地沙盒的视频文件进行展示和播放。
6. 在首页以纵向列表查看所有视频日记。
7. 在画廊视图查看所有视频日记。
8. 点进详情页查看完整视频和完整文字。

## 核心视图

### 首页视图

- 所有视频日记上下排列。
- 视频和搭配的文字必须放在一起展示。
- 首页只展示文字摘要。
- 点击可以进入详情视图。
- 上下滑动时视频不动；停止滑动后，只播放当前屏幕中最主要的一条可见视频。

### 画廊视图

- 展示已经加入的视频日记。
- 只展示文字摘要。
- 点击可以进入详情视图。
- 具体视觉布局以后续 Figma 指定内容为准。

### 详情视图

- 展示完整视频。
- 展示完整文字。
- 视频自动播放并循环播放。
- 支持再次编辑文字，入口由后续 Figma 指定。
- 支持删除视频日记，但只删除 App 本地记录和 App 沙盒视频文件，不删除系统相册原视频。

## 已确认决策

1. 视频导入后复制到 App 本地沙盒。
2. 文字保存在 App 本地数据库。
3. 摘要显示正文前面固定字数的一部分，具体字数和样式后续确认。
4. 导入后必须进入编辑页。
5. 首页同一时间只播放最主要的一条视频。
6. 支持再次编辑。
7. 支持删除，但任何时候都不删除系统相册原视频。
8. 最低支持 iOS 17.0。
9. 后续可能在第二版加入视频分类功能，第一版不实现。
10. 项目使用 GitHub 做版本管理，默认 `main` 为稳定分支，开发分支使用 `codex/` 前缀。

## 当前待确认事项

正式开发前需要确认：

1. 摘要固定字数、行数和截断样式。
2. 首页“最主要的一条视频”的判定方式。
3. 画廊布局和交互。
4. 再次编辑按钮的位置和样式。
5. 删除入口、确认流程和视觉样式。
6. 首页、画廊、详情、导入和编辑页面的具体 Figma 节点。
7. 视频是否静音自动播放，以及是否显示播放控制条。
8. 是否需要浅色和深色模式。
9. 账号、同步、备份、导出、搜索等后续能力是否进入未来版本。
