# vimember Roadmap

> 当前计划基于用户在 2026-05-24 给出的产品描述。正式制作前，所有视觉实现必须等待用户指定读取 Figma 内容。

## 状态说明

- `DONE`：已完成。
- `READY`：需求明确，可以进入下一步。
- `WAITING`：需要用户确认。
- `NOT STARTED`：尚未开始。

## Phase 1：制作前确认

状态：`READY`

目标：在开始写 App 前确认会影响架构和交互的关键问题。

已确认：

1. 最低 iOS 版本为 iOS 17.0。
2. 视频导入后复制到 App 沙盒。
3. 文字保存到 App 本地数据库。
4. 摘要显示正文前面固定字数的一部分，具体字数和样式后续确认。
5. 首页停止滚动后只播放当前屏幕中最主要的一条视频。
6. 导入后必须进入编辑页。
7. 支持再次编辑，按钮位置由 Figma 指定。
8. 支持删除，删除只影响 App 本地数据和 App 沙盒视频文件，不删除系统相册原视频。
9. 账号、同步、备份、导出等能力后续再说，MVP 不提前实现。
10. 第二版可能加入视频分类功能，第一版不实现。
11. 项目使用 GitHub 做版本管理，`main` 为稳定分支，开发分支使用 `codex/` 前缀。

仍需用户后续指定：

1. 首页视图的 Figma 节点。
2. 画廊视图的 Figma 节点。
3. 详情视图的 Figma 节点。
4. 导入和文字编辑流程的 Figma 节点。
5. 再次编辑和删除的按钮位置、确认流程和视觉样式。
6. 视频是否静音自动播放，以及是否显示播放控制条。

## Phase 2：项目脚手架

状态：`NOT STARTED`

目标：创建 iOS 项目基础结构。

计划内容：

1. 建立 Xcode 工程。
2. 建立 SwiftUI App 入口。
3. 建立基础目录结构。
4. 建立空的导航结构。
5. 设置最低支持 iOS 17.0。
6. 接入基础权限描述。
7. 建立 SwiftData 本地数据库基础模型。
8. 添加 App 沙盒视频文件目录约定。
9. 添加最小可构建验证。
10. 创建 GitHub 仓库并推送初始代码，按用户指令建立 `main` 和 `codex/` 开发分支。

验收：

1. App 能在模拟器或真机 Build。
2. 尚不实现完整业务。
3. 文档同步更新当前状态。

## Phase 3：视频导入与日记创建

状态：`NOT STARTED`

目标：完成从相册导入视频、添加文字、保存为视频日记的主流程。

计划内容：

1. 相册视频选择。
2. 视频资源读取并复制到 App 沙盒。
3. 文字输入。
4. 使用本地数据库保存视频日记记录。
5. 权限、取消、失败状态处理。
6. 保存原相册来源标识作为记录或未来关联。

依赖：

1. 导入和编辑页面 Figma 节点已指定并读取。

## Phase 4：首页视图

状态：`NOT STARTED`

目标：实现所有视频日记上下排列的首页体验。

计划内容：

1. 首页数据加载。
2. 视频和摘要文字组合展示。
3. 滚动时暂停或保持视频不运动。
4. 停止滚动后播放当前屏幕中最主要的一条可见视频。
5. 点击进入详情视图。

实现前需确认：

1. 首页 Figma 节点已指定并读取。
2. “最主要的一条视频”的判定方式已由用户确认。
3. 摘要字数和展示样式已由用户确认。

## Phase 7：删除、本地数据安全与第二版预留

状态：`NOT STARTED`

目标：实现只删除 App 本地内容的安全删除能力，并为第二版分类功能避免明显结构阻碍。

计划内容：

1. 删除本地数据库记录。
2. 删除 App 沙盒中的对应视频文件。
3. 确认删除不会调用删除系统相册原视频的 API。
4. 为未来分类功能保留可迁移的数据结构空间，但不实现分类 UI。
