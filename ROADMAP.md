# vimember Roadmap

> 当前计划基于用户在 2026-05-24 给出的产品描述和后续 Figma 读取结果。正式制作具体页面前，所有视觉实现必须读取用户指定的 Figma 内容；已读取并实现的内容需要在本文件中同步状态。

## 状态说明

- `DONE`：已完成。
- `IN PROGRESS`：已开始，已有可验证交付，但仍有核心内容未完成。
- `READY`：需求明确，可以进入下一步。
- `WAITING`：需要用户确认。
- `NOT STARTED`：尚未开始。

## Phase 1：制作前确认

状态：`IN PROGRESS`

目标：在制作具体页面和流程前确认会影响架构和交互的关键问题。

已确认：

1. 最低 iOS 版本为 iOS 17.0。
2. 视频导入后复制到 App 沙盒。
3. 文字保存到 App 本地数据库。
4. 摘要显示正文前面固定字数的一部分，具体字数和样式后续确认。
5. 首页停止滚动后只播放当前屏幕中最主要的一条视频。
6. 导入后必须进入编辑页。
7. 支持再次编辑，按钮位置由 Figma 指定。
8. 支持删除，删除只影响 App 本地数据和 App 沙盒视频文件，不删除系统相册原视频。
9. 账号、同步、备份、导出等能力后续再说，MVP 不提前实现。
10. 第二版可能加入视频分类功能，第一版不实现。
11. 项目使用 GitHub 做版本管理，`main` 为稳定分支，开发分支使用 `codex/` 前缀。
12. Figma 中 `首页` Frame 是首页内容来源，`首页-切换` Frame 是首页切换设置卡片来源。
13. 首页正文最多显示两行。
14. 首页左下角入口为搜索框，右下角入口为添加新内容，右上角入口为切换设置。
15. 首页视频下方需要纯色底、渐变模糊透明融合，并从视频模糊后区域动态取色。
16. 首页 `首页` Frame 和 `首页-切换` Frame 已读取。
17. Figma `首页` Frame 中的 `iPhone Air` 图层作为屏幕范围参考，基准尺寸为 `420pt * 912pt`。
18. 首页悬浮控件按真实设备屏幕尺寸计算，单位为 iOS point，优先使用 `UIScreen.main.bounds.size`。
19. 当前首页悬浮控件基准值已确认：右上切换按钮 `44pt * 44pt`，距右 `20pt`、距上 `69pt`；底部搜索框 `306pt * 50pt`，右下添加按钮 `50pt * 50pt`，间距 `10pt`，底部整体左右边距 `27pt`，距真实屏幕底部 `27pt`。

仍需用户后续指定：

1. 画廊视图的 Figma 节点。
2. 详情视图的 Figma 节点。
3. 导入和文字编辑流程的 Figma 节点。
4. 再次编辑和删除的按钮位置、确认流程和视觉样式。
5. 视频是否静音自动播放，以及是否显示播放控制条。
6. 首页日期显示方式切换规则。
7. 搜索框交互范围、结果页和空状态。

## Phase 2：项目脚手架

状态：`DONE`

目标：创建 iOS 项目基础结构。

已完成：

1. 使用 XcodeGen 建立 iOS 工程配置 `project.yml`。
2. 生成 `vimember.xcodeproj`。
3. 建立 SwiftUI App 入口 `vimember/App/vimemberApp.swift`。
4. 建立基础目录结构：`App`、`Domain`、`Features/Home`、`Services`、`Resources`。
5. 设置最低支持 iOS 17.0。
6. 接入相册权限描述 `NSPhotoLibraryUsageDescription`。
7. 建立本地开发脚本 `scripts/xcode-dev`，支持生成、构建、安装、启动和截图。
8. 添加首页演示用视频资源。
9. 添加最小可构建验证。

验收：

1. App 能在模拟器 Build：`DONE`。
2. 尚不实现完整业务：`DONE`。
3. 文档同步更新当前状态：`DONE`。

尚未完成或后续阶段处理：

1. SwiftData 持久化模型尚未接入，留到视频导入与日记创建阶段处理。
2. App 沙盒视频文件目录约定尚未落地，留到视频导入阶段处理。
3. GitHub 初始代码推送、分支和 PR 流程按用户后续明确指令处理。

## Phase 4：首页视图

状态：`IN PROGRESS`

目标：实现所有视频日记上下排列的首页体验。

已完成：

1. 已读取 Figma `首页` Frame 和 `首页-切换` Frame。
2. 已确认 Figma `首页` Frame 中 `iPhone Air` 图层作为屏幕范围参考，基准尺寸为 `420pt * 912pt`。
3. 已将首页悬浮控件换算规则写入指导文件：使用真实设备屏幕尺寸，单位 iOS point，优先以 `UIScreen.main.bounds.size` 计算。
4. 已实现首页演示数据列表 `HomeView`。
5. 已实现视频播放表面 `VideoPlayerSurface`，首页同一时间只播放当前选中的演示视频。
6. 已实现滚动停止后延迟选择当前主要可见视频的原型逻辑。
7. 已实现视频下方纯色底、底部模糊图像和渐变透明融合的原型效果。
8. 已实现从视频底部帧取平均色的异步采样与缓存服务 `VideoColorSampler`。
9. 已实现标题、日期、横线和正文结构，正文在首页最多显示两行。
10. 已实现左下搜索入口、右下添加入口、右上切换设置入口。
11. 已按最新 Figma 更新首页悬浮控件基准值：右上切换按钮 `44pt * 44pt`，距右 `20pt`、距上 `69pt`；底部搜索框 `306pt * 50pt`，右下添加按钮 `50pt * 50pt`，间距 `10pt`，底部整体左右边距 `27pt`，距真实屏幕底部 `27pt`。
12. 已实现 `首页-切换` 风格的切换设置卡片原型，包含日期入口和画廊入口的视觉占位。

剩余计划内容：

1. 首页数据加载。
2. 将首页演示数据替换为真实本地数据库数据。
3. 将首页视频资源替换为导入后复制到 App 沙盒的视频文件。
4. 结合真实数据和更多设备尺寸继续校准视频、纯色底、文字区与 Figma 的贴近度。
5. 完成滚动时暂停或保持视频不运动的最终行为验证。
6. 在用户确认“最主要的一条视频”判定方式后，固化当前屏幕主要可见视频算法。
7. 点击进入详情视图。
8. 接入搜索入口的真实交互。
9. 接入添加新内容入口的导入流程。
10. 日期显示方式切换规则确认后，再实现具体规则。
11. 画廊视图确认后，再接入切换到画廊视图。

继续实现前需确认：

1. “最主要的一条视频”的判定方式已由用户确认。
2. 日期显示方式切换规则已由用户确认。
3. 搜索框交互范围已由用户确认。
4. 导入、详情、画廊相关 Figma 节点已指定并读取。
